# 🌾 Projeto CanaVision Global Solution FIAP

Projeto acadêmico desenvolvido como parte da avaliação da Global Solution da FIAP. O trabalho consiste na estruturação de requisitos e na modelagem completa de um banco de dados relacional para sustentar uma plataforma AgTech voltada para o setor sucroenergético.

## 🎓 Contexto Acadêmico
A Global Solution é o principal desafio prático da FIAP exigindo a criação de soluções tecnológicas inovadoras para problemas reais. Neste cenário o CanaVision foi idealizado e documentado desde a sua concepção de negócios até a engenharia de dados. O projeto consolida o aprendizado prático em modelagem relacional, estruturação de entidades e regras de segurança da informação corporativa.

## 🎯 Visão do Negócio e Posicionamento
O CanaVision é uma plataforma de inteligência preditiva estruturada para lidar com o volume massivo de informações do campo. O projeto foca especificamente na estimativa de produtividade, mas também possui várias outras funções integradas dentro da sua arquitetura para cruzar variáveis climáticas e operacionais.

É fundamental destacar que o CanaVision foi concebido estritamente como um apoio ao agrônomo. Ele é feito para ajudar esse profissional no dia a dia do monitoramento. A tecnologia não atua sozinha e não vai substituir o agrônomo de forma alguma. Todo o ecossistema de dados visa organizar as informações para que o especialista humano tome a melhor decisão possível no gerenciamento da safra, respeitando totalmente a sua expertise no campo.

## ⚙️ Arquitetura e Engenharia de Dados
A sustentação de uma plataforma AgTech exige um repositório de dados capaz de garantir integridade e performance. A modelagem entregue na Global Solution conta com as seguintes características técnicas:

* **Escopo Estrutural:** O ambiente foi modelado com 34 entidades interligadas abrangendo desde o registro das lavouras até as métricas de estimativa de colheita.
* **Controle de Acesso RBAC:** Foi implementada uma arquitetura de segurança rigorosa baseada em Role Based Access Control. O modelo garante que apenas usuários com as permissões corretas acessem dados estratégicos da usina.
* **Integridade Relacional:** Aplicação de chaves primárias e estrangeiras respeitando as formas normais para evitar redundância na base de dados.
